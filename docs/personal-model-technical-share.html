开篇 · Personal Memory 是什么、为什么需要它

一句话：普通 Agent 只能看见当前会话；Personal Memory 让它在进入任务时，就知道当前进度、相关人物和用户反复表现出的工作方式，而且每一条判断都能回到证据。

先对齐几个术语

AX Tree 观测来源
macOS Accessibility 暴露的窗口、控件、焦点和文本结构。它不是 Memory。
Event 变化信号
窗口切换、输入、点击或内容变化，只说明“发生了变化”。
Capture 证据帧
系统在某个时刻保存的前台应用、焦点元素、可见文本和 URL。
Personal Memory 长期结构
从多次活动中形成的事实、关系和行为模式，可被检查、修正和召回。
Memory Formation 形成
Event 经过过滤、状态构造和证据门禁，逐步进入 Personal Memory。
Memory Recall 召回
当前情境从 Personal Memory 中选择相关部分，组织成 Agent Context。

为什么普通 Agent 不能直接接着昨天工作？

第二天，小张重新打开 Claude Code，继续处理昨天没有完成的项目 A。对一个普通 Agent 来说，这是一次新的会话。它只看得到当前仓库和小张刚刚输入的内容。

现在做到哪里？ 这件事正在和谁一起做？ 自己通常会怎样推进这类修改？
① 会话上下文会重置
昨天的任务进度和协作信息不会自动进入今天的 Claude Code。用户必须重复解释。
② 活动日志不等于个人结构
即使保存了昨天的全部屏幕内容，系统仍不知道哪些是项目状态、稳定关系或反复出现的做法。

Persome 怎么解决？

目标不是把更多日志塞进 Prompt，而是让活动先形成可以长期使用的结构，再由当前任务选择相关部分。

Memory Formation
App / AX Tree → Event → Capture → 当前状态 → Personal Memory
Memory Recall
当前情境 → Memory Attention Heads → 相关结构 → Agent Context

整条技术主线只做三件事

① 看清楚
从连续、重复、敏感的 Mac 事件中保留真正有意义的证据。
② 谨慎沉淀
LLM 可以提出候选，但事实、关系和行为模式必须经过门禁与重复证据。
③ 在正确时刻召回
当前项目、人物和任务决定哪些 Memory 应该进入 Agent Context。

0 · 走一个实例：Agent 最终应该知道什么

先不讲 Point、Line、Face，也不讲检索算法。先站在 Agent 的位置，确定最终需要拿到什么。

场景：第二天重新打开 Claude Code

小张再次进入项目 A 的仓库。Claude Code 此刻需要的不是昨天的完整流水，而是与当前工作直接相关的一块个人上下文：

项目 A 当前正在处理数据解析稳定性。昨天已经完成重试路径修复，
相关测试通过；接下来还需要补浏览器验证和完整测试。[事实引用 1]

小李是项目 A 的主要协作者，最近参与了数据契约和重试方案的讨论。
[事实引用 2]

小张在跨模块修改前通常先复现问题、补回归测试，再扩大修改范围。
[事实引用 3]

第一句话是刚刚发生、可以被验证的项目状态。第二句话来自多次协作。第三句话只有在多个任务中出现相同顺序后，才有资格成为模式。

近期事实（Fact）
某件具体事情当前是什么状态。
人物关系（Relation）
人与人、人与项目之间存在什么稳定联系。
行为模式（Behavior Pattern）
同一种做法是否跨任务反复出现。
事实引用
它不是第四种 Memory，而是每一句判断的证据入口。
Recall 中的一句话 → 对应的 Fact / Relation / Behavior Pattern → 支撑它的一个或多个低层 Fact → 每个 Fact 对应的 Raw Evidence
[事实引用 1] 近期项目事实 
Fact
  项目 A 的重试路径已完成修复，相关测试通过。

Raw Evidence
  [Claude Code · owner-authored]
  “重试路径已经修复，相关测试通过。”

  [Terminal · tool result]
  “tests passed”

Evidence Scope
  当前项目 A Session，重试路径相关窗口
[事实引用 2] 人物与项目关系 
Relation
  self knows 小李，label = collaborator
  self participates_in 项目 A
  小李 participates_in 与项目 A 相关的多个 Activity

Supporting Facts
  小李参与了项目 A 的数据契约讨论。
  小李与小张共同检查了重试方案。
  小李在多个项目 A Session 中持续出现。

Raw Evidence
  [Chat · 小李 received]
  “数据契约部分我来一起看。”

  [Claude Code · owner-authored]
  “和小李对齐重试方案后继续修改。”
[事实引用 3] 行为模式 
Behavior Pattern
  小张在跨模块修改前通常先复现问题并补回归测试。

Supporting Facts
  项目 A：先复现重试失败，再修改实现。
  项目 B：先补解析回归测试，再调整数据结构。
  项目 C：先运行完整门禁，再扩大重构范围。

Raw Evidence
  每条 Supporting Fact 分别引用对应 Session 中的小张输入、
  测试命令与结果，以及最终完成状态。
当前实现边界
Memory Delta 已保存 Quote、Session 和时间窗口，但 Point 没有直接保存精确 capture_id，Timeline 也只保存 capture_count。逐句 Fact 到 Raw Capture 的 lineage 仍需补齐。
